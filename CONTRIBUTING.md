# Contributing to Xaviour AI Demo

Thanks for your interest in contributing to Xaviour.

This repository is a **public demo** of the Xaviour product experience. It is meant to showcase the interface, interaction model, and product direction using mock data and simulated behavior.

## What Contributions Are Welcome

We welcome thoughtful contributions that improve the public demo, including:

- UI polish and accessibility improvements
- bug fixes
- performance improvements
- demo-data improvements
- documentation improvements
- code quality improvements
- responsive behavior and visual consistency
- safe developer experience improvements

## What Is Out of Scope

Please do not submit pull requests that attempt to add or recreate:

- private production prompts
- proprietary AI orchestration logic
- real Gmail or calendar integrations
- real OAuth flows
- hidden backend systems
- internal product strategy material
- production secrets or infrastructure assumptions

The goal of this repository is to remain a **safe public demo**, not the full private product.

## Before You Start

Please:

1. read the README
2. check existing issues or discussions first
3. keep changes focused and easy to review
4. preserve the premium Xaviour product direction
5. avoid broad redesigns unless explicitly discussed

## Development

Typical local setup:

```bash
npm install
npm run dev
```

## Contribution Guidelines

- keep pull requests focused and reasonably small
- explain the problem and the proposed solution clearly
- prefer incremental improvements over large rewrites
- preserve the current product story and design language
- avoid introducing unnecessary dependencies
- keep demo behavior mock-only and public-safe

## Design / UX Principle

Xaviour should feel:

- calm
- premium
- editorial
- intelligent
- low-noise
- product-led

If you are changing UI, preserve that direction.

## Security

If your finding is security-sensitive, do **not** open a public issue.
Please use the process described in `SECURITY.md`.
