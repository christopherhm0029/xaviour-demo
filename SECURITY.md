# Security Policy

Thank you for helping keep Xaviour AI safe.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately by email:

**christopher@xaviour.ai**

Please include:

- a clear description of the issue
- steps to reproduce it
- affected files, routes, or components
- any proof of concept if available
- your contact details for follow-up

## Please Do Not

- open a public GitHub issue for sensitive security reports
- publicly disclose the vulnerability before it has been reviewed
- include secrets, personal data, or unnecessary sensitive information in public channels

## Scope

This repository is a **public demo** and uses mock/demo functionality. Even so, security issues are taken seriously, especially if they could affect:

- users running the demo locally
- public deployments of the demo
- supply chain or dependency safety
- accidental exposure of sensitive information

## Response Approach

We will aim to:

- acknowledge receipt of valid reports
- review the issue as quickly as possible
- determine severity and impact
- fix or mitigate confirmed issues in a reasonable timeframe

## Safe Harbor

Good-faith security research intended to improve the safety of this demo will be treated respectfully.

Thank you for reporting issues responsibly.
